## 02. 첫 번째 윈도우 창 만들기

1. **QApplication (심장):** 프로그램 그 자체입니다. 초기화 설정, 테마 관리, 그리고 **이벤트 루프**를 담당합니다. 프로그램 당 딱 **하나**만 있어야 합니다.
2. **QWidget (세포):** 
3. **Event Loop (영혼):** 이게 가장 중요합니다.
   - 



### 1. 핵심 개념 3가지

#### **1) QApplication**: 프로그램의 "사령탑"

- 프로그램 그 자체입니다. 
- 모든 PySide6 프로그램에는 **단 하나**의 `QApplication` 객체만 있어야 합니다.
- 초기화 설정, 테마 관리, 그리고 **이벤트 루프**를 담당합니다. 
- 이벤트 처리, 시스템 설정 등을 총괄하는 **<u>매니저</u>** 역할

#### **2) QWidget**: 화면에 보이는 "창"

- 화면에 보이는 모든 것이 '위젯'입니다. 버튼, 입력창, 그리고 우리가 띄울 '창(Window)' 자체도 위젯입니다.
- 버튼, 텍스트 등 모든 UI 요소의 부모 클래스
- 빈 창부터 시작해서 점차 위젯을 추가하는 방식

#### **3) Event Loop (이벤트 루프)**: 프로그램이 "살아있게" 하는 심장

- 일반적인 파이썬 스크립트는 코드가 끝나면 바로 종료되죠?
- GUI 프로그램은 종료되면 안 됩니다. 사용자가 버튼을 누를 때까지 **'무한 대기'**해야 합니다. 
  - `app.exec()` 구문이 실행되면 프로그램이 종료되지 않고 대기 상태로 들어간다.

- 이 '대기 상태'를 유지하고 사용자의 행동(클릭, 키보드 입력)을 감시하는 녀석이 바로 이벤트 루프입니다.
  - 사용자의 클릭, 키보드 입력 등을 계속 감지하며 반응함

### 2. 실전 코드

#### 1) 실습 1: 가장 간단한 윈도우

이제 텍스트 에디터(VS Code, PyCharm 등)를 열고 `main.py`를 만들어서 아래 코드를 작성해보세요. **복사 붙여넣기 보다는 직접 타이핑하며 흐름을 느껴보세요.**

```python
# main.py
import sys
from PySide6.QtWidgets import QApplication, QWidget

# 1. QApplication 객체 생성 (필수!)
# sys.argv는 명령줄 인수를 받기 위함입니다.
app = QApplication(sys.argv)

# 2. 윈도우(QWidget) 생성 (화면에 보일 요소)
window = QWidget()
window.setWindowTitle("나의 첫 PySide6 프로그램")  # 창 제목
window.setGeometry(100, 100, 400, 300)  # (x, y, 너비, 높이)

# 3. 윈도우 화면에 표시
# 메모리에만 있던 위젯을 실제 모니터에 그려줍니다.
window.show()

# 4. 이벤트 루프 시작 (프로그램이 종료되지 않고 대기)
# 사용자가 창을 닫을 때까지 이 라인에서 멈춰있게 됩니다.
sys.exit(app.exec())
```

**[ 코드 분석 ]**

| 코드                      | 의미                               |
| ------------------------- | ---------------------------------- |
| `QApplication(sys.argv)`  | 프로그램 초기화 (명령줄 인자 전달) |
| `QWidget()`               | 빈 윈도우 객체 생성                |
| `setWindowTitle()`        | 창 제목 설정                       |
| `setGeometry(x, y, w, h)` | 창 위치(x,y)와 크기(w,h) 설정      |
| `show()`                  | 창을 화면에 표시                   |
| `app.exec()`              | 이벤트 루프 시작 (여기서 멈춤!)    |

- `app = QApplication(sys.argv)`:
  - Qt 프로그램을 시작하기 위한 필수 의식입니다. `sys.argv`를 넘겨주는 이유는 터미널에서 실행할 때 옵션을 받을 수 있게 하기 위함입니다.
- `window.show()`:
  - 이걸 호출하지 않으면 창은 만들어졌지만(메모리 상에 존재), 우리 눈에는 보이지 않습니다. (Invisible 상태)
- `app.exec()`:
  - 이 메서드가 호출되는 순간 프로그램은 **무한 루프(Event Loop)**에 빠집니다. "사용자가 X 버튼을 누를 때까지 기다려!"라고 명령하는 겁니다.
- `sys.exit(...)`:
  - 프로그램이 종료될 때 운영체제에게 "나 정상적으로 종료됐어(0)" 혹은 "에러가 났어(1)"를 알려주기 위해 감싸줍니다.





#### 2) 실습 2: 클래스 기반 구조 (권장 방식)

```python
# main.py
import sys
from PySide6.QtWidgets import QApplication, QWidget

class MyWindow(QWidget):
    def __init__(self):
        super().__init__()
        self.initUI()
    
    def initUI(self):
        # 윈도우 설정
        self.setWindowTitle("클래스 기반 윈도우")
        self.setGeometry(300, 300, 500, 400)

if __name__ == "__main__":
    app = QApplication(sys.argv)
    window = MyWindow()
    window.show()
    sys.exit(app.exec())
```

**💡 왜 클래스를 사용할까?**

- 코드 재사용성 ↑
- 확장성 ↑ (나중에 기능 추가가 쉬움)
- 프로젝트가 커져도 관리하기 쉬움

## 03 Qt Designer 소개

코드로만 UI를 짜다 보면 "버튼 위치를 픽셀 단위로 계산하기 너무 귀찮은데?"라는 생각이 들 겁니다. 이때 사용하는 것이 **Qt Designer**입니다.

- **역할:** 포토샵처럼 드래그 앤 드롭으로 버튼과 창을 배치할 수 있는 그래픽 툴입니다.
- **파일 형식:** 여기서 작업한 결과물은 `.ui` 확장자로 저장됩니다.
- **위치:** `pip install PySide6`를 했다면 보통 파이썬 설치 폴더의 `Lib/site-packages/PySide6` 안에 `designer.exe`라는 이름으로 숨어 있습니다. (찾기 힘들다면 `pyside6-designer` 명령어로 실행될 수도 있습니다.)

> **교수의 조언:**
>
> "시니어로서 조언하자면, 처음에는 **Qt Designer를 쓰지 마세요.**"
>
> 코드로 직접 위젯을 배치해봐야 레이아웃(Layout)의 원리를 이해할 수 있습니다. Designer는 그 원리를 이해한 뒤에 생산성을 높이기 위해 쓰는 도구입니다. 당분간은 손코딩으로 갑시다.



### 1) Qt Designer란?

- **드래그 앤 드롭**으로 UI를 디자인할 수 있는 시각적 도구
- `.ui` 파일로 저장되며, Python 코드로 변환 가능



### 2) Qt Designer 실행하기

```bash
# 설치 확인
designer

# 또는 직접 경로 실행 (Windows 예시)
C:\Python311\Lib\site-packages\PySide6\designer.exe
```



### 3) 간단한 사용법

1. **File → New → Widget** 선택
2. 왼쪽 위젯 목록에서 드래그해서 배치
3. **File → Save As** → `my_window.ui` 저장
4. Python 코드로 변환:

```bash
pyside6-uic my_window.ui -o ui_my_window.py
```

5. Python에서 불러오기:

```python
from ui_my_window import Ui_Form

class MyWindow(QWidget):
    def __init__(self):
        super().__init__()
        self.ui = Ui_Form()
        self.ui.setupUi(self)
```

**📌 현재 단계에서는**: Qt Designer를 "이런 게 있구나" 정도만 알고 넘어가자. 코드로 직접 작성하는 게 개념 이해에 더 좋아!

## 04. 요약 및 실습 과제

### 1. 핵심 요약

| 개념             | 역할                 | 코드                           |
| ---------------- | -------------------- | ------------------------------ |
| **QApplication** | 프로그램 관리자      | `app = QApplication(sys.argv)` |
| **QWidget**      | 화면에 보이는 창     | `window = QWidget()`           |
| **Event Loop**   | 프로그램을 계속 실행 | `app.exec()`                   |
| **show()**       | 창을 화면에 표시     | `window.show()`                |



### 2. 실습 과제

1. ✅ 창 제목을 "Hello PySide6"로 설정
2. ✅ 창 크기를 800x600으로 변경
3. ✅ 창 위치를 화면 중앙에 배치 (힌트: `QApplication.primaryScreen()`)

```python
# 힌트: 화면 중앙 배치
screen = QApplication.primaryScreen().geometry()
x = (screen.width() - window.width()) // 2
y = (screen.height() - window.height()) // 2
window.move(x, y)
```



### 3. 자주 묻는 질문

**Q1: `sys.exit(app.exec())`가 뭐야?**

- `app.exec()`: 이벤트 루프 시작 (프로그램이 여기서 "대기")
- `sys.exit()`: 이벤트 루프 종료 시 프로그램을 깔끔하게 종료

**Q2: `super().__init__()`는 왜 필요해?**

- 부모 클래스(`QWidget`)의 초기화 코드를 실행하기 위함
- 이게 없으면 Qt의 내부 설정이 안 돼서 오류 발생!

**Q3: 창이 깜빡이고 사라져요!**

- `window.show()` 이후 `app.exec()`를 호출했는지 확인
- 이벤트 루프가 없으면 프로그램이 바로 종료됨
